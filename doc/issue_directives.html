function issue_directives

Handling the scribejs, issue X,Y,Z type directives. The method returns a set of strings to be added to the final (markdown) minutes.

While the comments are invisible when displayed as part of the minutes, they may be used by postprocessing tools that gathers a portion of the discussions that is relevant to a particular issue or set of issues.

If the original directive just includes a - instead of the issue numbers a special HTML/markdown comment returned of the form <!-- issue - -->. This may be used as a "closing" part by a post-processing step to cut off the text in the rest of a (sub)section from being an issue comment.

Parameters

🔗
config: Configuration
  • the general scribejs configuration object
🔗
directive: string
  • either 'pr' or 'issue'
🔗
issue_references: string
  • comma separated list of issue numbers, possibly of the form repo#number

Return Type

🔗
IssueInformation

the markdown lines to be added to the final minutes
